DHCP: Dynamic Host Configuration Protocol:


Dynamic Host Configuration Protocol, or DHCP, is a client/server protocol that assigns ip addresses to devices in a network. Devices can have different IP addresses each time they connect to the network; no address is

fixed to a particular device permanently. Not only this, but devices only have a particular IP address for a certain length of time (called a lease). Devices can contact the DHCP server and renew their IP address if they wish.

DHCP makes adding new devices to a network simpler since there is no need for an administrator to uniquely assign each new device a unique IP address. Any changes in the network are handled easily by the DHCP server.


When a client joins a network running a DHCP server, it sends a discover broadcast around the network to find the server. Once the server is discovered, it randomly chooses an available IP address and sends ARP request packets around the network to ensure that the IP address hasn't already been taken. If it doesn't receive an ARP reply, it sends a UDP "offer" packet to the new host containing the IP address. The client then sends a UDP "request" packet back to the DHCP server, requesting can it use that IP. Finally, the DHCP server replies with a UDP "acknowledge" packet, and the client is assigned the IP address and given a lease. After the lease is up, a client can renew its IP address by repeating the same process. The DHCP server may offer it the IP address it previously had, or it may offer the client a new one.



To simulate this using netanim, go to the draw tab and add a single router. Add three hosts to the network and connect each to the router. Each link should have delay 100, bandwith 100 and a loss of 0. Once all this has been added, hit the save button and then press the "run dhcp" option. This will set up your custom topology in Mininet but each host will have no IP address. In order to assign each host an IP address, each host must connect to the DHCP server that is running on the router. Go to the host CLI and type:


dhclient h1-eth0 -v


Dhclient will now contact the DHCP server and print out each step as it occurs, displaying the configuration settings it received. Do this for each host.

Once each host has an IP address, you can send traffic along the network as normal. Try sending a ping between two of the hosts.

Once you're finished with mininet, type exit. Now load the output xml file into the animator and watch the simualtion, paying close attention to the UDP packets that appear, as well as the ARP packets.
